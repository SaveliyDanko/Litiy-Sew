# Feature flags

Feature-флаги управляют тем, какие части сайта собираются и показываются клиенту.
Они читаются из переменных окружения Vite (`import.meta.env.*`) в модуле
[frontend/src/utils/featureFlags.ts](../frontend/src/utils/featureFlags.ts).

## VITE_SHOP_ENABLED

Включает магазинную часть сайта. Когда флаг выключен, сайт работает в режиме
портфолио — бэкенд Spring Boot вообще не требуется.

| Значение | Поведение |
| --- | --- |
| `true` | Доступны все разделы: главная, «Обо мне», коллекции, каталог выкроек, корзина, избранное, профиль, оформление заказа, юридические страницы. |
| `false` (по умолчанию) | Доступны только: главная, «Обо мне», коллекции (`/collections`, `/collections/*`), юридические страницы (`/legal/*`). Магазинные URL (выкройки, корзина, избранное, профиль, оформление заказа) отвечают 404. Пункт «Выкройки», иконки корзины/избранного/профиля, блок «Новинки» на главной и подвал (Footer) не отображаются. |

Если переменная не задана, считается, что флаг `false`.

### Как включить / выключить магазин

1. В каталоге [frontend/](../frontend/) создайте файл `.env.local` (или отредактируйте существующий). Пример есть в [frontend/.env.example](../frontend/.env.example).
2. Укажите значение флага:

   ```bash
   # включить магазин
   VITE_SHOP_ENABLED=true

   # выключить — остаётся только портфолио
   VITE_SHOP_ENABLED=false
   ```

3. Перезапустите dev-сервер (`npm run dev`) или пересоберите прод-бандл (`npm run build`). Vite подставляет значение `import.meta.env.*` на этапе сборки, поэтому без перезапуска изменение не подхватится.

### Что именно скрывается при `VITE_SHOP_ENABLED=false`

- **Маршруты** (отдают страницу 404): `/patterns`, `/patterns/*`, `/cart`, `/favorites`, `/auth`, `/profile`, `/profile/*`, `/checkout`.
- **Шапка** ([Header.tsx](../frontend/src/components/Header.tsx)): пункт меню «Выкройки» и иконки избранного / корзины / профиля.
- **Подвал** ([Footer.tsx](../frontend/src/components/Footer.tsx)): компонент возвращает `null` и не рендерится ни на одной странице. Ссылки на юридические страницы остаются доступными по прямому URL (`/legal/*`).
- **Главная** ([HeroSection.tsx](../frontend/src/components/HeroSection.tsx)): блок «Новинки» скрыт, карточка коллекции остаётся.

Коллекции (`/collections`, `/collections/*`) и юридические страницы (`/legal/privacy`, `/legal/terms`, `/legal/offer`) остаются доступными в обоих режимах — они не зависят от backend и относятся к портфолио-части сайта.

### Когда backend не нужен

В режиме `VITE_SHOP_ENABLED=false` фронтенд не делает запросов к Spring Boot API — можно не поднимать [backend/](../backend/) и связанный `docker-compose` (PostgreSQL). Для обратного режима backend обязателен, иначе страницы магазина будут получать сетевые ошибки.

### Добавление новых флагов

1. Добавьте запись в [frontend/src/utils/featureFlags.ts](../frontend/src/utils/featureFlags.ts) через хелпер `readFlag`.
2. Добавьте переменную в [frontend/.env.example](../frontend/.env.example) с комментарием и значением по умолчанию.
3. Опишите флаг в этом файле в отдельной секции по аналогии с `VITE_SHOP_ENABLED`.
